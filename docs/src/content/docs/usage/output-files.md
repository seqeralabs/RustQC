---
title: Output Files
description: Reference for all output files generated by RustQC, including dupRadar analysis files and featureCounts-compatible outputs.
---

RustQC generates two categories of output files in a single pass: **dupRadar
analysis** files and **featureCounts-compatible** files. All outputs use the
BAM file stem as a prefix (e.g., `sample.bam` produces `sample_dupMatrix.txt`).

Each output can be individually enabled or disabled via the
[configuration file](/RustQC/usage/configuration/).

## dupRadar outputs

### Duplication matrix

**File:** `<sample>_dupMatrix.txt`

A tab-separated file with one row per gene and 14 columns. This is the core
data table from which all other dupRadar outputs are derived.

| Column | Description |
|--------|-------------|
| `ID` | Gene identifier from the GTF |
| `geneLength` | Gene length in base pairs (non-overlapping exon bases) |
| `allCountsMulti` | Total read count including multi-mappers |
| `filteredCountsMulti` | Read count excluding duplicates, including multi-mappers |
| `dupRateMulti` | Duplication rate with multi-mappers: `(all - filtered) / all` |
| `dupsPerIdMulti` | Number of duplicate reads (multi-mapper inclusive) |
| `RPKMulti` | Reads per kilobase (multi-mapper inclusive) |
| `RPKMMulti` | Reads per kilobase per million mapped reads (multi-mapper inclusive) |
| `allCounts` | Total read count (unique mappers only) |
| `filteredCounts` | Read count excluding duplicates (unique mappers only) |
| `dupRate` | Duplication rate (unique mappers only) |
| `dupsPerIdUnique` | Number of duplicate reads (unique mappers only) |
| `RPK` | Reads per kilobase (unique mappers only) |
| `RPKM` | Reads per kilobase per million mapped reads (unique mappers only) |

Genes with zero reads have a duplication rate of `0` and RPK/RPKM values of `0`.
The format is identical to R dupRadar's `_dupMatrix.txt` output.

### Intercept and slope

**File:** `<sample>_intercept_slope.txt`

A two-line text file containing the intercept and slope of the fitted logistic
regression model:

```
intercept	0.03186
slope	1.60189
```

These parameters define the fitted curve: `y = 1 / (1 + exp(-(intercept + slope * x)))`,
where `x` is log10(reads/kbp) and `y` is the duplication rate.

### Density scatter plot

**Files:** `<sample>_duprateExpDens.png`, `<sample>_duprateExpDens.svg`

The primary diagnostic plot showing duplication rate vs. expression level, with
point density encoded as color and the fitted logistic curve overlaid in red.
See [Interpreting Plots](/RustQC/guide/interpreting-plots/) for details.

### Duplication rate boxplot

**Files:** `<sample>_duprateExpBoxplot.png`, `<sample>_duprateExpBoxplot.svg`

Box-and-whisker plot showing the distribution of duplication rates across
expression-level bins. See [Interpreting Plots](/RustQC/guide/interpreting-plots/)
for details.

### Expression histogram

**Files:** `<sample>_expressionHist.png`, `<sample>_expressionHist.svg`

Histogram of gene expression levels (log10 reads/kbp) across all genes.
See [Interpreting Plots](/RustQC/guide/interpreting-plots/) for details.

### MultiQC files

**Files:**

- `<sample>_intercept_slope_mqc.tsv` -- Intercept and slope values in MultiQC
  general statistics format.
- `<sample>_duprateExpDensCurve_mqc.txt` -- Fitted duplication rate curve data
  points for MultiQC line plot visualization.

These files are designed for integration with
[MultiQC](https://multiqc.info/), enabling dupRadar results to be included in
multi-sample QC reports.

## featureCounts-compatible outputs

These files are generated in the same pass as the dupRadar analysis. There is
no need to run featureCounts separately.

### Counts file

**File:** `<sample>.featureCounts.tsv`

A tab-separated counts file compatible with the
[Subread featureCounts](http://subread.sourceforge.net/) output format. Contains
per-gene read counts with annotation columns:

| Column | Description |
|--------|-------------|
| `Geneid` | Gene identifier |
| `Chr` | Chromosome(s) |
| `Start` | Start position(s) |
| `End` | End position(s) |
| `Strand` | Strand(s) |
| `Length` | Gene length |
| `<sample>` | Read count for this sample |

The file includes a header comment line with the command used to generate it.

### Summary file

**File:** `<sample>.featureCounts.tsv.summary`

Assignment summary statistics in featureCounts format, reporting the number of
reads in each category:

- `Assigned` -- reads successfully assigned to a gene
- `Unassigned_NoFeatures` -- reads not overlapping any gene
- `Unassigned_Ambiguity` -- reads overlapping multiple genes

### Biotype counts

**File:** `<sample>.biotype_counts.tsv`

A tab-separated file with per-biotype read counts, sorted alphabetically by
biotype name:

```
protein_coding	12345
lncRNA	678
rRNA	90
```

The biotype is extracted from the GTF attribute specified by `biotype_attribute`
(default: `gene_biotype` for Ensembl, `gene_type` for GENCODE).

### Biotype MultiQC files

**Files:**

- `<sample>.biotype_counts_mqc.tsv` -- Biotype counts formatted as a MultiQC
  bargraph data file, suitable for visualizing the distribution of reads across
  biotypes.
- `<sample>.biotype_counts_rrna_mqc.tsv` -- rRNA percentage formatted as a
  MultiQC general statistics value, reporting the fraction of assigned reads
  mapping to rRNA genes.
