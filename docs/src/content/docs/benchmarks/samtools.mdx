RustQC produces samtools-compatible output files (flagstat, idxstats, stats SN section)
and RSeQC-compatible TIN (Transcript Integrity Number) analysis as part of its
single-pass BAM processing. This page compares the output of each tool against the
originals on a 10 GB paired-end BAM file (GM12878, ENCODE).

## Performance

All samtools-compatible outputs and TIN analysis run together in the same
single-pass BAM reading that also produces dupRadar, featureCounts, RSeQC, preseq,
and gene body coverage output.

| Tool | Traditional | RustQC |
| --- | ---: | ---: |
| samtools flagstat | ~1m | -- |
| samtools idxstats | ~1s | -- |
| samtools stats | ~5m | -- |
| RSeQC tin.py | ~45m | -- |
| **All outputs (single pass)** | -- | **5m 13s** |

### Minor differences

| Metric | samtools | RustQC | Notes |
| --- | ---: | ---: | --- |
| bases mapped (cigar) | 17,220,065,351 | 17,218,285,734 | 0.01% difference in CIGAR base counting |

| insert size average | 1553.3 | 3084.6 | Different outlier filtering strategy |
| insert size stdev | 2276.6 | 15028.6 | Different outlier filtering strategy |

The insert size differences are due to samtools applying iterative outlier
filtering (removing pairs beyond mean +/- N*stdev, then recomputing), while
RustQC uses all properly paired reads. Both approaches are valid; the insert
size statistics are informational rather than used for downstream analysis.

The output format includes the `samtools stats` header comment required for
MultiQC parsing.

## TIN (Transcript Integrity Number)

RustQC's TIN implementation produces per-gene TIN scores that measure transcript
integrity via Shannon entropy of read coverage uniformity.

### Summary statistics

| Metric | RustQC |
| --- | ---: |
| Genes analysed | 16,499 |
| TIN mean | 72.55 |
| TIN median | 83.89 |
| TIN stdev | 26.14 |

The TIN analysis completes as part of the single-pass BAM processing. The
original RSeQC `tin.py` requires a separate full BAM pass and can take 45+
minutes for a 10 GB file.

### Output format

RustQC produces gene-level TIN scores (one row per gene, using the longest
transcript as representative), while RSeQC's `tin.py` produces transcript-level
scores. Both formats are compatible with MultiQC. The gene-level approach
provides a cleaner summary with one score per gene.

## Gene body coverage (Qualimap)

RustQC produces Qualimap-compatible gene body coverage output, including:
- A 100-bin coverage profile along normalized gene body positions (5' to 3')
- A Qualimap `rnaseq_qc_results.txt` file parseable by MultiQC

### Coverage bias metrics

| Metric | Value |
| --- | --- |
| 5' bias | 1.20 |
| 3' bias | 0.92 |
| 5'-3' bias | 1.24 |

These metrics help identify degradation patterns -- a 5'-3' bias significantly
above 1.0 may indicate RNA degradation, while values near 1.0 indicate
uniform coverage.

## Benchmark conditions

- **BAM file**: GM12878 ENCODE RNA-Seq, paired-end, ~185M reads, 10 GB
- **samtools version**: 1.23 (via Docker)
- **RSeQC version**: 5.0.4 (via Docker)
- **RustQC**: 10 threads, `--gtf` mode
- **Hardware**: Apple M3 Max, 128 GB RAM
