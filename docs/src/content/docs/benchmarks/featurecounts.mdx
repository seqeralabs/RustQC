RustQC includes built-in gene-level read counting that produces output
compatible with the Subread featureCounts format. This page compares the
counting performance and output accuracy against standalone Rsubread
featureCounts.

Benchmarks were run on AWS cloud infrastructure on 2026-03-09.

## Performance

### Large dataset

**Large benchmark input:** GM12878 REP1 -- a paired-end RNA-seq BAM aligned to
GRCh38 (~186M reads).

| Tool                            | Wall time | CPU time | Peak RSS |
| ------------------------------- | --------: | -------: | -------: |
| Rsubread featureCounts          |     3m 3s |     3m 3s | 210 MB  |
| RustQC (all tools, single pass) |   41m 24s |  41m 24s |  11.4 GB |

RustQC's single pass includes featureCounts-compatible counting alongside
dupRadar duplication analysis, all 8 RSeQC tools (including TIN), preseq, samtools
outputs, and gene body coverage profiling. The traditional workflow requires
running each tool separately.

### Small dataset

**Small benchmark input:** test sample -- ~52K reads, chr6 only.

| Tool                            | Wall time | CPU time | Peak RSS |
| ------------------------------- | --------: | -------: | -------: |
| Rsubread featureCounts          |     3m 9s |       1s |   3.1 MB |
| RustQC (all tools, single pass) |    3m 31s |    25.9s | 182.1 MB |

The RustQC wall time includes loading the GTF annotation and running all QC
analyses in a single pass; the 1s CPU time for featureCounts reflects that the
small dataset is trivially sized and most wall time is container startup.

## Output equivalence

RustQC's read counting uses the same algorithm as Subread featureCounts:

- **Feature type:** exon-level features grouped by `gene_id`
- **Overlap detection:** at least 1 base pair overlap
- **Strand awareness:** configurable via the `-s` / `--strandedness` flag
- **Multi-mapping:** tracked separately for unique and multi-mapper columns

counts match exactly for every gene. The output format is directly compatible
with downstream tools such as DESeq2 and MultiQC.

## Additional outputs

Beyond the standard featureCounts counts file and summary, RustQC also
produces:

- **Biotype counts** (`.biotype_counts.tsv`) -- per-biotype read count
  summaries
- **Biotype MultiQC bargraph** (`.biotype_counts_mqc.tsv`) -- ready for
  MultiQC visualization
- **rRNA percentage** (`.biotype_counts_rrna_mqc.tsv`) -- rRNA fraction for
  MultiQC general statistics

Generating these in the traditional workflow requires additional scripting after
the featureCounts run.
