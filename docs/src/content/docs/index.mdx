## Why RustQC?

<CardGrid stagger>
  <Card title="Identical Results" icon="approve-check">
    Produces bit-for-bit identical output to R's dupRadar and featureCounts.
    Over 800,000 values compared with zero mismatches.
  </Card>
  <Card title="9x Faster" icon="rocket">
    Processes a 10 GB BAM file in 3 minutes (single thread) vs 30 minutes for
    the R equivalent. Multi-threaded mode brings this down to under 1 minute.
  </Card>
  <Card title="All-in-One" icon="puzzle">
    Combines dupRadar analysis, featureCounts-compatible output, and biotype
    counting in a single pass — no need to run featureCounts separately.
  </Card>
  <Card title="SAM / BAM / CRAM" icon="document">
    Accepts SAM, BAM, and CRAM alignment files. Process multiple files in
    parallel with automatic thread distribution.
  </Card>
</CardGrid>

## Credits

RustQC is a reimplementation of the excellent [dupRadar](https://bioconductor.org/packages/dupRadar/)
Bioconductor package by Sergi Sayols, and uses the same counting approach as
[Subread featureCounts](https://subread.sourceforge.net/) by Yang Liao and Wei Shi.

**If you use RustQC, please cite the original tools** — see the [Credits & Citation](/about/credits/) page.
